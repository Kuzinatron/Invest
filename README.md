# Презентация: Численные методы решения систем линейных алгебраических уравнений

## Тема проекта

Разработка интерактивной презентации по численным методам решения СЛАУ

## Требования к выполнению

### Общие требования

- Презентация должна быть выполнена с помощью Markdown, где это возможно
- Можно выполнить на любом понравившемся фреймворке или написать собственный
- Тема (оформление) обязательно должна быть вынесена в отдельный CSS файл
- Для написания сценария разрешается использовать любой удобный фреймворк
- Минимальное количество слайдов: 20

### Структурные требования

- Перед каждым разделом необходимо использовать слайд-разделитель
- Оформление презентации должно быть читаемым (например, черно-синий текст на черном фоне не допускаются)
- Обязательными являются титульный слайд и заключительный слайд
- Обязательно наличие меню для быстрого перехода к слайду
- Обязательно наличие навигации: стрелки, номер слайда

### Содержательные требования

1. **Титульный слайд** - с указанием темы, автора, группы
2. **Определение численных методов**
3. **Определение СЛАУ**
4. **Решение СЛАУ методом Гаусса** - формулы, алгоритм и пример
5. **Решение СЛАУ методом Якоби** - формулы, алгоритм и пример
6. **Невязка** - определение и вычисление
7. **Интерактивная программа** - написанная на языке JavaScript, решающая СЛАУ выбранными методами
8. **Исходный код программы**
9. **Сравнение методов** - преимущества и недостатки
10. **Заключительный слайд** - выводы

### Требования к программе

- Оформление ввода/вывода необходимо сделать с помощью LaTeX
- Допускается использование библиотеки mathjs для работы с математическими выражениями
- Программа должна поддерживать оба метода решения (Гаусса и Якоби)
- Должен вычисляться показатель точности решения (невязка)

### Дополнительные слайды

- При наличии оставшихся слайдов, но не менее 3, заполнить веселыми картинками по теме

### Технические требования

- Все файлы проекта должны быть в одной папке
- Презентация должна открываться в браузере без установки дополнительного ПО
- Код должен быть хорошо структурирован и содержать комментарии
